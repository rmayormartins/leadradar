Parâmetros da varredura

A chave roda só no seu navegador, direto contra a API do Google. Nada é enviado a outro servidor. Veja "como configurar" abaixo.
O botão WA de cada empresa abre o WhatsApp com esta mensagem já preenchida. {link} vira automaticamente o modelo do nicho da empresa (barbearia, dentista ou mecânica) e, para os demais nichos, o portfólio geral. Personalize antes de enviar: abordagem individual converte mais e evita bloqueio por spam.
Como configurar (5 a 10 min)
  1. Acesse console.cloud.google.com, crie um projeto e ative o faturamento (o Google exige cartão mesmo para uso dentro da faixa gratuita).
  2. Em APIs e serviços > Biblioteca, ative a Places API (New). Atenção: é a versão "New", a antiga virou legada.
  3. Em Credenciais, crie uma chave de API. Em restrições, escolha Sites e adicione https://seuusuario.github.io/* e, para testes locais, http://localhost:8000/*. Em restrições de API, marque apenas a Places API (New).
  4. Publique este arquivo num repositório com GitHub Pages (pode renomear para index.html). Para testar local, rode python -m http.server na pasta e abra localhost:8000.
  5. Cole a chave no campo acima e inicie a varredura.

Cota: cada requisição desta ferramenta usa o Text Search na categoria Enterprise (é ela que devolve telefone e site). A faixa gratuita atual é de cerca de 1.000 requisições por mês, e cada requisição traz até 20 empresas. O contador no topo mostra quanto você já gastou na sessão.

Importante: nada fica salvo entre sessões. Exporte o CSV antes de fechar a página. Use os contatos para abordagens individuais e personalizadas, sem disparo em massa.
